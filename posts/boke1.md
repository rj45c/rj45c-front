# 前端博客搭建初体验：从本地调试到线上部署
大家好～ 这是我的第一篇技术博客，主要目的是验证本地开发环境与线上部署环境的正常运行，同时记录下前端博客搭建的一些基础步骤和踩坑点，希望能给同样刚入门的小伙伴提供一点参考。
作为一名前端初学者，一直想拥有一个属于自己的技术博客，既能沉淀学习笔记，也能锻炼自己的文档撰写能力。经过一番调研，最终选择了基于Hexo搭建静态博客，搭配GitHub Pages完成线上部署，整个过程不算复杂，但也遇到了几个小问题，在这里一一记录下来。
## 一、本地环境搭建核心步骤
本地环境是博客开发的基础，只有本地调试无误，才能顺利部署到线上。核心依赖主要有Node.js、Git和Hexo，三者的安装和配置有几个关键注意点，避免踩坑。
### 1.1 依赖安装注意事项
首先需要安装Node.js，建议选择LTS版本（长期支持版），避免使用最新版导致部分插件不兼容。安装完成后，通过命令行输入node -v和npm -v验证是否安装成功，确保版本对应（Node.js 14+即可）。
其次是Git的安装，主要用于后续代码提交和部署，安装完成后配置全局用户名和邮箱，方便后续与GitHub关联。最后通过npm安装Hexo-cli，命令为npm install -g hexo-cli，安装完成后输入hexo -v验证。
### 1.2 本地博客初始化与调试
依赖安装完成后，新建一个文件夹作为博客根目录，进入目录后执行hexo init初始化博客，初始化完成后会生成默认的目录结构和文件。接着执行hexo s启动本地服务器，访问http://localhost:4000即可看到博客的默认页面。
这里需要注意，若启动失败，大概率是端口被占用，可通过hexo s -p 5000指定其他端口，或者关闭占用4000端口的进程后重新启动。
## 二、线上部署流程简化
本地调试无误后，就可以部署到线上了，这里选择GitHub Pages，操作简单且免费，适合新手。核心步骤分为两步：创建GitHub仓库和配置部署参数。
### 2.1 GitHub仓库创建
登录GitHub，新建一个仓库，仓库名称必须为「用户名.github.io」（例如我的用户名为test，仓库名称就是test.github.io），否则无法正常访问。仓库创建完成后，复制仓库的SSH地址，用于后续与本地博客关联。
### 2.2 部署配置与执行
回到本地博客根目录，修改_config.yml文件中的deploy配置，填写仓库SSH地址和分支（默认main或master），配置如下：
deploy:
  type: git
  repo: git@github.com:用户名/用户名.github.io.git
  branch: main
配置完成后，安装部署插件npm install hexo-deployer-git --save，然后执行hexo clean && hexo g && hexo d，依次完成清理、生成静态文件和部署操作。
## 三、双环境验证结果与总结
部署完成后，等待1-5分钟，访问「用户名.github.io」即可看到线上博客页面，与本地调试页面一致，说明本地+线上双环境运行正常，本次测试圆满完成。
### 3.1 测试总结
本次测试主要验证了Hexo博客的本地搭建和线上部署流程，核心要点的是依赖版本兼容、配置参数正确，以及部署命令的顺序执行。虽然过程中遇到了端口占用、部署失败等小问题，但通过查阅文档和排查，都顺利解决了。
